Pipes

Tutaj rozumiane jako te zmienianie formatu wyświetlanych danych, raczej nie będzie tu nic o pipe jako metodzie do Observable

Przykładowe zastosowania

{{ 'Zmienia na wielkie litery' | uppercase }}

{{ 'ZMIENIA NA MAŁE LITERY' | lowercase }}

formatuje date {{ testDate | date: 'fullDate' }}

Wbudowane pipes w Angularze

Chaining pipes

Możemy łączyć pipes aby osiągnąć wynik w jednej linii.
wykonywane są od lewej do prawej

przykład:

{{ testDate | date: 'fullDate' | uppercase }}

Custom pipe

możemy tworzyć customowe. Trzeba pamiętać aby zachwować zasadę pipe i nie zmieniać w transform pierwotnych danych.

Customowe pipe też deklarujemy w ngModule

Przykłady:

{{ testCustomPipe1 | pokemon }}

{{ testCustomPipe2 | parametrize:1:'kto czytał ten trąba' }}

Ciekawsze zastosowania pipes (pure i impure pipes)

ustawienie atrybutu pure na false sprawia, że pipe będzie aktywowany za każdym razem jak dane do których jest podłączony zostaną zmienione. Defaultowo jest ustawiony na true.

Przykład:


{{service.category}}

{{service.name}}

Async Pipe

wbudowany pipe który po tym jak Promise albo observable zostanie resolve wyświetla dane

Przykład:

{{ asyncPipeTestPromise | async }}

{{ asyncPipeTestObservable | async }}
